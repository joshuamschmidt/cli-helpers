daily note structure in obsidian
--- date: 2026-03-20 --- ## Todo - [ ] fix token refresh race condition ^todo-a3f2 - [ ] review PR #448 ^todo-b1c9 - [x] update staging config ^todo-d7e1 ## Sessions ### fix-auth (todo-a3f2) **09:14 → 11:42 · 2h 28m** Intent: token refresh is failing under concurrent load... Commands: 47

Block references (^todo-xxxx) are the key — the REST API can PATCH exactly that list item without touching anything else in the file.

todo item states
pending - [ ]Created, not started. No session attached yet.
active - [/]wj start called. Session clock running.
done - [x]wj stop --done. Duration written inline.
paused - [-]wj stop --pause. Time accumulates on next start.

An additional "rolled over" state copies pending/paused items to the next day's note, preserving accumulated time on paused items.

new commands
wj todo add "description" [--tag x]
Generates a short block ID (todo-xxxx). Appends to today's daily note under ## Todo. Also writes to ~/.wj/todos/YYYY-MM-DD.todo as local fallback. Prints the ID.
wj todo ls [--pending | --done | --all]
Lists todos from today's daily note or local file. Shows ID, state badge, time spent if completed.
wj start <todo-id | name>
If argument matches a todo ID, loads that todo's description as the session label and marks item [/] in Obsidian. Plain names still work — fully backward compatible.
wj stop [--done | --pause]
--done (default) marks [x] with duration inline. --pause marks [-] and preserves accumulated time for next session on the same ID.
wj todo rollover
Copies all pending/paused items from yesterday into today's note. Preserves accumulated time. Auto-runs on first wj command of the day if configured.
the REST API calls (inside obsidian.sh)
_wj_obs_todo_start() { local block_id="$1" local date_path="Daily/$(date +%Y-%m-%d).md" curl -ks -X PATCH \ -H "Authorization: Bearer $OBSIDIAN_API_KEY" \ -H "Operation: replace" \ -H "Target-Type: block" \ -H "Target: $block_id" \ -H "Content-Type: text/plain" \ --data "- [/] ${_WJ_TODO_DESC} ^${block_id} " \ "https://127.0.0.1:27124/vault/${date_path}" } _wj_obs_todo_done() { local block_id="$1" duration="$2" curl -ks -X PATCH \ -H "Authorization: Bearer $OBSIDIAN_API_KEY" \ -H "Operation: replace" \ -H "Target-Type: block" \ -H "Target: $block_id" \ -H "Content-Type: text/plain" \ --data "- [x] ${_WJ_TODO_DESC} ^${block_id} " \ "https://127.0.0.1:27124/vault/${date_path}" }
local-first design — obsidian is a view, not a database
Every todo operation writes to ~/.wj/todos/YYYY-MM-DD.todo first, then mirrors to Obsidian if $OBSIDIAN_API_KEY is set and the API responds. The local file is always source of truth. This means the system works identically with Obsidian closed — the GUI, graph links, and Dataview queries are a bonus, not a dependency.
